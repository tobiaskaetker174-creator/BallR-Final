BallR — Product & Feature Documentation
Version: Current (March 2026) | Stack: React Native + Expo Router + Supabase

1. What Is BallR
BallR is a mobile-first pickup football app for expat and local football communities in Southeast Asia. Players discover and book into organised 5v5–10v10 casual matches at local venues, build a persistent skill rating (ELO), compete in monthly city leaderboards, and earn rewards for top performance. The app targets Bangkok and Bali as its first two cities.

Core value proposition:

Find a game today, pay online, show up
Build a real skill rating across dozens of games
Know who your best teammates are
Win the month → get free games + a t-shirt

6. Core Business Logic
6.1 ELO Rating System
File: src/lib/elo.ts

Standard Elo with K-factor = 32, range 500–2500.

Expected score  = 1 / (1 + 10^((opponentElo - playerElo) / 400))
ELO change      = round(32 × (actual − expected))
  actual: win=1.0, draw=0.5, loss=0.0

No-show penalty: −50 ELO (floored at 500).

Team balancing algorithm (balanceTeams):

Sort all players by ELO descending
Snake draft: rounds alternate Blue-then-Red, Red-then-Blue
Up to 5 swap-improvement passes: try every Blue↔Red swap, keep if it reduces |avgBlue − avgRed|
ELO tiers (getEloTier):

Range	Tier	Icon
≥1500	Elite	👑
≥1300	Expert	💎
≥1100	Advanced	🔥
≥900	Intermediate	⚡
≥700	Beginner	🌱
<700	Novice	⚽
Win probability: round(expectedScore(teamA, teamB) × 100) — displayed on game detail screen.

6.2 Dignity / Rating Protection System
File: src/lib/dignity.ts

Prevents discouraging new/weak players by hiding their exact ELO.

Rules:

Calibrating: games_played < 10 → show "Casual ⚽" instead of ELO
Protected: ELO ≤ city's 50th percentile (only if city has ≥10 players; otherwise threshold = 900) → show "Casual ⚽"
Detailed stats visible: only if NOT protected
Leaderboard visible: NOT protected AND NOT banned AND elo_calibrated = true
This means roughly the bottom ~50% of players in each city never see a number — just "Casual ⚽". This is intentional to keep the community friendly.

6.3 Player of the Month Scoring
File: src/lib/potm.ts

POTM score is a weighted composite (0–100):

Component	Weight	Input
Win Rate	35%	wins / gamesPlayed × 100
ELO	25%	normalised from 800–1400 range → 0–100
Peer Skill Rating	10%	normalised 1–5 → 0–100
Peer Sportsmanship	10%	normalised 1–5 → 0–100
Community Quality	7%	normalised 1–10 → 0–100 (default 50 if none)
Community Sportsmanship	3%	same
Reliability	10%	reliability_score (already 0–100)
Eligibility thresholds:

Minimum 5 games played (POTM_MIN_GAMES = 5)
Minimum 3 peer ratings received (POTM_MIN_RATINGS = 3)
Minimum 70% reliability score (POTM_MIN_RELIABILITY = 70)
Top 10 shown (POTM_TOP_N = 10)
Must pass shouldShowOnLeaderboard() (see Dignity system)

6.6 Formation Assignment
File: src/lib/formation.ts

Assigns players to positions on each team for the visual formation display.

Player score = elo_rating + (avg_community_quality_rating ?? avg_skill_rating ?? 3) × 50

Algorithm:

Fill GK slot first — highest-scoring preferred GK player
Fill DEF → MID → FWD in order, preferring players with matching preferred_positions, falling back to highest-scoring available
Leftover players appended to FWD
Formation scales with team size:

Players	DEF	MID	FWD
5	2	2	1
6	2	2	2
8	3	3	2
10	3	3	3
Each assigned player has a reasoning string (e.g. "Preferred MID (1240 ELO)" or "Flexible – assigned DEF (980 ELO)") shown in the UI.
